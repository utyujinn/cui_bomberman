# Bitmap Editor

このツールはAIで生成しました。

8x8から最大64x32までのビットマップ画像を編集できるCUI型のドットエディタです。`▀`（上半ブロック文字）を使用して、文字色と背景色で2ピクセルを表現します。

## 特徴

- **可変キャンバスサイズ**: 8x8から64x32まで自由に設定可能
- **RGBカラーピッカー**: 直感的なバー操作でRGB値を調整
- **16進数カラー入力**: `#FF00AA`のような形式で色を直接指定
- **ファイル読み込み**: C言語の配列形式のファイルから読み込み
- **配列エクスポート**: C言語のコードとして出力可能
- **プリセットパレット**: 10色のカラーパレット搭載

## コンパイル

```bash
cd bitmap

# ビットマップエディタ
cc editor.c -o editor

# BMP→配列変換ツール
cc bmptoarray.c -o bmptoarray
```

## 使い方

### 基本的な起動

```bash
# デフォルト（8x8）
./editor

# サイズ指定（幅 x 高さ）
./editor 16 16

# ファイルを読み込んで起動
./editor 8 8 color.txt

# 大きなキャンバス
./editor 32 16
```

## 操作方法

### 通常モード

| キー | 機能 |
|------|------|
| `W` / `S` / `A` / `D` | カーソル移動（上/下/左/右） |
| `SPACE` | 現在の色でペイント |
| `I` | スポイト（カーソル位置の色を取得） |
| `C` | カラーパレットから色を選択（循環） |
| `R` | RGBカラーピッカーモードに入る |
| `H` | 16進数で色を入力 |
| `T` | 透過モードをトグル |
| `L` | ファイルから配列を読み込み |
| `E` | C言語配列形式でエクスポート |
| `Q` | 終了 |

### RGBカラーピッカーモード

| キー | 機能 |
|------|------|
| `W` / `S` | チャンネル選択（R/G/B） |
| `A` / `D` | 値を減少/増加（5ステップ） |
| `ENTER` | 確定して通常モードに戻る |
| `Q` | キャンセル |

## ファイル形式

### 入力ファイル（例: color.txt）

以下のようなC言語の配列形式のファイルを読み込めます：

```c
int texture[8][8] = {
    {0x202020, 0x202020, 0x202020, 0x202020, 0x202020, 0x202020, 0x202020, 0x202020},
    {0x202020, 0xFF0000, 0x00FF00, 0x0000FF, 0xFFFF00, 0xFF00FF, 0x00FFFF, 0x202020},
    {0x202020, 0x202020, 0x202020, 0x202020, 0x202020, 0x202020, 0x202020, 0x202020},
    {0x202020, 0xFFFFFF, 0x808080, 0x000000, 0xA00000, 0x00A000, 0x0000A0, 0x202020},
    {0x202020, 0x202020, 0x202020, 0x202020, 0x202020, 0x202020, 0x202020, 0x202020},
    {0x202020, 0x202020, 0x202020, 0x202020, 0x202020, 0x202020, 0x202020, 0x202020},
    {0x202020, 0x202020, 0x202020, 0x202020, 0x202020, 0x202020, 0x202020, 0x202020},
    {0x202020, 0x202020, 0x202020, 0x202020, 0x202020, 0x202020, 0x202020, 0x202020}
};
```

- サイズは自動検出されます
- 16進数カラーコード（0xRRGGBB形式）
- 最大64x32まで対応

### エクスポート形式

`E`キーを押すと、現在のキャンバス内容をC言語の配列形式で出力します。この出力をそのまま`graphics.c`などに貼り付けて使用できます。

```c
int texture[8][8] = {
    {0x202020, 0x202020, 0x202020, 0x202020, 0x202020, 0x202020, 0x202020, 0x202020},
    // ... 以下省略
};
```

## カラーパレット

デフォルトで以下の10色が用意されています（`C`キーで循環）：

1. 黒 (`0x000000`)
2. 白 (`0xFFFFFF`)
3. 赤 (`0xFF0000`)
4. 緑 (`0x00FF00`)
5. 青 (`0x0000FF`)
6. 黄 (`0xFFFF00`)
7. マゼンタ (`0xFF00FF`)
8. シアン (`0x00FFFF`)
9. グレー (`0x808080`)
10. ダークレッド (`0xA00000`)

## 技術詳細

### 表示方法

`▀`（上半ブロック、Unicode U+2580）を使用して、1文字で縦2ピクセルを表現します：

- **文字色（foreground）**: 上のピクセルの色
- **背景色（background）**: 下のピクセルの色

これにより、ターミナル上で効率的にドット絵を描画できます。

### カーソル表示

カーソル位置は黄色で強調表示されます：
- 上半分にカーソルがある場合：文字色が黄色
- 下半分にカーソルがある場合：背景色が黄色

### 透過機能

透過モード（`T`キーでトグル）をONにすると、描画したピクセルに透過フラグ（`0x01000000`）が設定されます：

- **透過フラグ**: 色値の最上位ビット（bit 24）を使用
- **チェッカーボード表示**: 透過ピクセルは灰色のチェッカーボード模様で表示されます
- **エクスポート形式**: 透過ピクセルは`0x01RRGGBB`形式で出力されます
- **用途**: `graphics.c`でスプライト描画時に背景を上書きしないために使用

例：

```c
// 通常のピクセル
0xFF0000  // 赤（不透明）

// 透過ピクセル
0x01FF0000  // 赤（透過）
```

## 制限事項

- 最大キャンバスサイズ: 64x32ピクセル
- ターミナルがANSIエスケープシーケンスとTrueColor（24bit色）に対応している必要があります
- Unicode文字`▀`が正しく表示できるフォントが必要です

## 使用例

### 1. 8x8のアイコンを作成

```bash
./editor 8 8
# WASDで移動、SPACEでペイント
# Cで色を変更しながら描画
# Eでエクスポート
```

### 2. 既存のテクスチャを編集

```bash
./editor 8 8 color.txt
# color.txtを読み込んで編集
# Eでエクスポートして上書き保存
```

### 3. 大きなバナーを作成

```bash
./editor 64 16
# 64x16の大きなキャンバスで作業
# RGBピッカーで細かい色調整
```

### 4. BMPファイルから配列を作成

```bash
# BMPファイルを配列に変換
./bmptoarray sprite.bmp sprite.txt

# エディタで読み込んで編集
./editor 8 8 sprite.txt
```

### 5. 透過付きスプライトを作成

```bash
./editor 8 8
# WASDで移動、スペースでペイント
# Tキーを押して透過モードをONにする（画面に[TRANSPARENT]と表示される）
# 透過にしたい部分をペイント（チェッカーボード模様で表示される）
# Tキーをもう一度押して透過モードをOFFに戻す
# 通常のペイントを続ける
# Eキーでエクスポート
```

## BMP→配列変換ツール (bmptoarray)

BMPファイル（24bit/32bitカラー）をC言語の配列形式に変換するユーティリティです。

### コンパイル

```bash
cc bmptoarray.c -o bmptoarray
```

### 使い方

```bash
# 基本的な使い方（標準出力）
./bmptoarray input.bmp

# ファイルに保存
./bmptoarray input.bmp output.txt

# リダイレクトで保存
./bmptoarray sprite.bmp > sprite.h
```

### 対応形式

- **24ビットBMP**: RGB各8ビット（一般的な形式）
- **32ビットBMP**: RGBA各8ビット（アルファチャンネルは無視）
- **ボトムアップ/トップダウン**: 両方に対応

### 出力形式

```c
// Generated from sprite.bmp
// Size: 16x16
int texture[16][16] = {
    {0xFF0000, 0x00FF00, 0x0000FF, ...},
    {0xFFFF00, 0xFF00FF, 0x00FFFF, ...},
    ...
};
```

### 使用例

#### 1. フォトショップ/GIMPで作成したBMPを変換

```bash
# 1. 画像編集ソフトで8x8のBMPファイルを作成
# 2. BMPとして保存（24bit推奨）
# 3. 配列に変換
./bmptoarray icon.bmp icon.txt

# 4. エディタで読み込み
./editor 8 8 icon.txt
```

#### 2. 既存のBMPスプライトをゲームで使用

```bash
# BMPを配列に変換してヘッダーファイルに保存
./bmptoarray player.bmp > player_sprite.h

# graphics.cなどにインクルード
```

#### 3. 大きな画像を分割して処理

```bash
# 64x32のBMPを変換
./bmptoarray banner.bmp banner.txt

# エディタで微調整
./editor 64 32 banner.txt
```

### ワークフロー例

完全なワークフローの例：

```bash
# 1. 画像編集ソフトでBMPを作成
#    例: GIMP, Photoshop, Paint.NETなど
#    サイズ: 8x8、16x16、32x32など
#    形式: 24bit BMP

# 2. BMPを配列に変換
./bmptoarray mysprite.bmp mysprite.txt

# 3. エディタで微調整
./editor 8 8 mysprite.txt
#    - 色の調整
#    - ピクセル単位の修正

# 4. エクスポート（Eキー）してコピー

# 5. graphics.cに貼り付け
```

## トラブルシューティング

### 色が正しく表示されない
- ターミナルがTrueColor（24bit色）に対応しているか確認してください
- 最新のターミナルエミュレータ（iTerm2, GNOME Terminal, Windows Terminalなど）を使用してください

### `▀`が正しく表示されない
- ターミナルのフォントがUnicodeブロック要素に対応しているか確認してください
- 等幅フォント（monospace）を使用してください

### ファイルが読み込めない
- ファイルパスが正しいか確認してください
- ファイルの配列形式が正しいか確認してください（`int texture[Y][X]`形式）
